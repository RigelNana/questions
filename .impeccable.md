# Impeccable Design Context

## Design Context

### Users
- **目标用户**: 准备后端/DevOps/SRE 面试的工程师（1-5年经验为主）
- **使用场景**: 碎片化时间刷题、系统性复习知识点、面试前集中突击
- **核心任务**: 高效地检验和巩固后端基础设施相关知识，发现知识盲区

### Brand Personality
- **三个词**: 专注 (Focused)、精准 (Precise)、高效 (Efficient)
- **语调**: 专业但不死板，像一个资深工程师朋友在帮你复习
- **情感目标**: 让用户感到掌控感和进步感，而非焦虑

### Aesthetic Direction
- **视觉基调**: Notion 风格的极简文档感 — 大量留白、清晰的层级、柔和的边界
- **主题**: 浅色主题为主
- **字体**: 系统字体栈，中文优先；代码块使用等宽字体（JetBrains Mono / Fira Code）
- **色彩策略**:
  - 主色: 中性蓝灰 (#37352F Notion 风格的深色文字)
  - 强调色: 柔和蓝 (#2383E2) 用于交互元素
  - 正确: 柔和绿 (#0F7B6C)
  - 错误: 柔和红 (#EB5757)
  - 背景: 温暖白 (#FFFFFF 主体, #F7F6F3 侧边栏)
  - 分割线/边框: #E9E9E7
- **反面参考**: 不要像传统在线教育平台那样花哨；不要游戏化元素堆砌
- **间距**: 8px 基础网格，宽松的行高(1.6-1.8)，段落间距充足
- **圆角**: 3-4px（Notion 风格的微圆角）

### Design Principles
1. **内容为王 (Content First)** — 一切设计服务于题目内容的可读性，题目文本、代码块、解释说明必须清晰可辨
2. **渐进呈现 (Progressive Disclosure)** — 答案和解释不一次性展开，用户先思考再揭晓，减少认知负担
3. **零摩擦导航 (Zero-Friction Navigation)** — 分类、筛选、进度一目了然，3次点击内到达任何题目
4. **无状态纯净 (Stateless Purity)** — 纯前端、无后端依赖，localStorage 持久化，题目数据可插拔 JSON 配置
5. **代码优先排版 (Code-First Typography)** — 代码片段、命令行、配置文件必须有一流的排版体验，支持语法高亮

### Language
- 界面语言: 中文为主
- 技术术语: 保留英文原文（如 Kubernetes、Redis、Goroutine 等）
- 代码和命令: 全英文
