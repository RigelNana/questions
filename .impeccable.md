# Impeccable Design Context

## Design Context

### Users
- **目标用户**: 准备后端/DevOps/SRE 面试的工程师（1-5年经验为主）
- **使用场景**: 碎片化时间刷题、系统性复习知识点、面试前集中突击
- **核心任务**: 高效地检验和巩固后端基础设施相关知识，发现知识盲区
- **使用情绪**: 沉浸和专注 — 像在读技术文档一样自然

### Brand Personality
- **三个词**: 专业 (Professional)、冷静 (Calm)、高效 (Efficient)
- **语调**: 专业但不死板，像一个资深工程师朋友在帮你复习
- **情感目标**: 让用户沉浸在知识中，自然流畅地阅读和理解

### Aesthetic Direction
- **视觉基调**: Linear/Arc 风格的冷色调 — 深蓝灰背景、毛玻璃模糊、微妙的光影层次
- **主题**: 深色/浅色双模式（`[data-theme="dark"]`），冷灰调为主
- **字体**:
  - 正文: LXGW WenKai (CDN: cdn.jsdelivr.net/npm/lxgw-wenkai-webfont)
  - 代码: JetBrains Mono (Google Fonts)
  - LXGW WenKai 同时覆盖中文和西文正文
- **色彩策略**:
  - 浅色主色: #FAFBFC (背景), #F0F2F5 (次级背景), #1A1D23 (文字)
  - 深色主色: #0B0D12 (背景), #12141A (次级背景), #E4E5E9 (文字)
  - 强调色: #6366F1 (Indigo, 浅色) / #818CF8 (深色)
  - 正确: #10B981 / #34D399
  - 错误: #EF4444 / #F87171
  - 警告: #F59E0B / #FBBF24
  - 边框: #E2E5EB (浅色) / #1E2130 (深色)
- **反面参考**: 不要像传统在线教育平台那样花哨；不要游戏化元素堆砌
- **间距**: 8px 基础网格，宽松的行高(1.6-1.8)，段落间距充足
- **圆角**: rounded-xl (12px) 为主要容器，rounded-lg (8px) 为按钮和输入框

### Design Principles
1. **内容为王 (Content First)** — 一切设计服务于题目内容的可读性，题目文本、代码块、解释说明必须清晰可辨
2. **渐进呈现 (Progressive Disclosure)** — 答案和解释不一次性展开，用户先思考再揭晓，减少认知负担
3. **零摩擦导航 (Zero-Friction Navigation)** — 分类、筛选、进度一目了然，3次点击内到达任何题目
4. **无状态纯净 (Stateless Purity)** — 纯前端、无后端依赖，localStorage 持久化，题目数据可插拔 JSON 配置
5. **代码优先排版 (Code-First Typography)** — 代码片段、命令行、配置文件必须有一流的排版体验，JetBrains Mono + 语法高亮

### Visual Effects
- **毛玻璃**: 固定元素 (Header, BottomNav, Sidebar) 使用 `backdrop-blur(12px) + saturate(180%)` (`.glass-surface`)
- **动画**: 使用 `animate-fade-in`, `animate-slide-up`, `animate-scale-in`，150-300ms 时长
- **骨架屏**: 加载状态使用 shimmer 渐变动画（`.skeleton` 类）
- **悬浮效果**: 卡片 hover 时 border 变为 accent 色 + 微弱 shadow

### CSS Architecture
- Tailwind v4 `@theme {}` 定义所有 design tokens（CSS 变量名保持 `--color-notion-*` 前缀以保持兼容性）
- 深色模式通过 `[data-theme="dark"]` 选择器切换
- 所有组件使用 `var(--color-notion-*)` 引用颜色，改变 CSS 变量即可全局换色

### Language
- 界面语言: 中文为主
- 技术术语: 保留英文原文（如 Kubernetes、Redis、Goroutine 等）
- 代码和命令: 全英文
